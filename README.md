# VibeCodingLed

VibeCodingLed 是一个 AI 编程状态提示灯。把产品插到电脑 USB 口后，AI 工具会通过配套 skill 自动完成检测和配置，之后你可以通过 LED 的亮灭和闪烁状态，快速知道 AI 编程任务是否正在运行、是否有结果需要查看。

## 基本原理

VibeCodingLed 通过 USB 同时完成供电和通信。电脑上的 AI 工具读取配套 skill 后，会把当前 AI 编程任务状态发送给状态灯；状态灯内部芯片收到状态后，用板载 LED 显示结果。

LED 有四种客户可见状态：

| LED 效果 | 含义 |
| --- | --- |
| 灯灭 | 当前没有需要监控或确认的 AI 编程任务 |
| 亮 1 秒、灭 1 秒 | 所有被监控的 AI 编程任务都在运行 |
| 亮 3 秒、灭 1 秒 | 部分任务已经完成或停止，但还有其它任务正在运行 |
| 常亮 | 所有被监控的任务都已经完成或停止，需要查看最终结果 |

常亮不一定代表失败，也可能表示任务已经正常完成。你看完结果后，可以让 AI 清除提醒并继续监控。

## 你会收到什么

| 物品 | 作用 |
| --- | --- |
| VibeCodingLed 实物产品 | 插到电脑 USB 口，用 LED 显示 AI 编程状态 |
| VibeCodingLed skill | 给 AI 工具读取，用于自动完成设备检测、配置和状态监控 |

请保存完整 skill 文件夹。不要只保存其中某一个文件，否则 AI 工具可能无法完成自动配置。

## 使用步骤

1. 把 VibeCodingLed 插到电脑上能传数据的 USB 口。
2. 下载并解压随产品提供的 VibeCodingLed skill。
3. 打开 Codex、Cursor 或其它支持读取 skill 的 AI 编程工具。只安装 Codex 也可以使用。
4. 让 AI 工具读取 VibeCodingLed skill。
5. 对 AI 工具说：

```text
请读取 VibeCodingLed USB 状态灯 skill，帮我自动检测设备、配置 AI 编程状态监控，并验证灯灭、亮1秒灭1秒、亮3秒灭1秒、常亮四种状态。
```

6. 等待 AI 工具完成配置，并按提示观察 LED 状态。

USB 口必须能通信，不只是供电。简单判断方法是：这个 USB 口平时能插 U 盘、键盘或鼠标。只用于充电的 USB 电源口不能使用。

## 如何判断已经正常工作

配置完成后，可以按下面现象判断：

| 现象 | 说明 |
| --- | --- |
| AI 告诉你设备已识别 | 电脑已经能和状态灯通信 |
| 没有需要监控或确认的任务时灯灭 | 当前没有任务需要关注 |
| 所有任务运行时亮 1 秒、灭 1 秒 | 状态监控已经开始工作 |
| 部分任务完成、部分仍运行时亮 3 秒、灭 1 秒 | 可以先查看已完成任务，也可以让其它任务继续跑 |
| 所有任务完成后常亮 | 有最终结果或问题需要你查看 |

如果插上后完全没有反应，先换一个能插 U 盘的数据 USB 口，再让 AI 重新读取 skill 检测。

## 简单演示

配置完成后，不要用“直接控制 LED”的方式演示。正确的演示目标是：确认 LED 是否跟随 AI 软件的真实任务状态变化。

### 演示 1：没有任务时灯灭

关闭或结束当前 AI 编程任务，等待几秒。此时 LED 应该熄灭，表示当前没有 AI 编程任务在运行。

### 演示 2：一个任务运行时灯闪烁

新建一个 AI 对话，发送一个稍微需要时间的简单任务，例如：

```text
请帮我写一份“如何整理电脑文件”的分步骤说明，要求至少 20 条，每条都写清具体操作。
```

AI 开始生成内容时，LED 应该亮 1 秒、灭 1 秒。这个现象说明状态灯已经能感知 AI 软件正在工作。

### 演示 3：部分任务停止时亮 3 秒、灭 1 秒，全部完成后常亮

亮 3 秒、灭 1 秒必须满足两个条件：

1. 当前 AI 工具能把多个任务的开始和停止状态提供给 VibeCodingLed。Windows + Codex 已支持。
2. 有一个任务已经停止，同时还有另一个任务仍在运行。

如果你使用 Codex，可以这样做：

1. 新建第一个 Codex 对话，发送一个较长任务，例如：

```text
请写一份 Python 入门学习计划，至少 120 条，每条都要有具体练习任务。
```

2. 在第一个对话还在生成时，新建第二个 Codex 对话，发送一个较短任务，例如：

```text
请列出 10 个整理电脑文件的建议。
```

3. 观察 LED：

| 过程 | LED 应该如何显示 |
| --- | --- |
| 两个 Codex 对话都在运行 | 亮 1 秒、灭 1 秒 |
| 短任务先完成，长任务仍在运行 | 亮 3 秒、灭 1 秒 |
| 所有 Codex 任务都结束后 | 常亮 |
| 你对 AI 说“我看完了，继续监控”后 | 如果没有其它运行任务，灯灭；如果还有任务运行，继续按剩余任务显示 |

如果你当前已经有其它 Codex 后台对话仍在运行，最后一步可能不会常亮，而是继续按剩余任务显示，这是正常的。

### 我已经看到提醒，如何继续监控

如果有 3 个任务同时运行，A 已经结束，B 和 C 还在运行，LED 会亮 3 秒、灭 1 秒，提醒你可以查看 A 的结果。产品本体没有确认按钮，确认入口是你正在使用的 AI 工具。你看完 A 后，不需要拔插产品，也不需要停止 B 和 C，直接对 AI 说：

```text
我已经知道刚才完成的任务了，请清除已结束任务提醒，继续监控剩余任务。
```

AI 会通过 skill 清除已经停止的提醒，只保留仍在运行的任务。此时如果 B 和 C 还在运行，LED 会回到亮 1 秒、灭 1 秒；之后 B 或 C 再结束时，LED 会再次变成亮 3 秒、灭 1 秒。等所有任务都结束后，LED 会常亮，直到你再次确认。

更短的说法也可以：

```text
我看完了，继续监控。
```

也可以直接让 AI 帮你组织这个测试：

```text
请根据 VibeCodingLed skill，判断当前 AI 工具是否能检测多个任务的开始和停止。如果可以，请帮我创建一个长任务和一个短任务，用它们验证“两个任务都运行时亮1秒灭1秒，一个任务停止、另一个任务仍运行时亮3秒灭1秒，所有任务都完成时常亮”。
```

这个演示能证明：状态灯不是只会亮灯，而是能够在 AI 工具提供可靠状态时，反映多个任务的运行和停止状态。

## 适用范围

当前至少支持 Windows 上 Codex 单独使用；Cursor 已提供接入方式。其它 AI 编程工具如果支持读取 skill，或能让 AI 在任务开始、停止时触发本地状态更新，也可以接入。

VS Code 本身是编辑器环境，不等同于 AI 任务状态源；在 VS Code 中使用 Codex 时，状态来自 Codex。macOS、Linux 等环境的后台启动和自动监控需要继续适配；产品本身仍然通过 USB 与电脑通信。

# VibeCodingLed

VibeCodingLed 是一个 AI 编程状态提示灯。把产品插到电脑 USB 口后，AI 工具会通过配套 skill 自动完成检测和配置，之后你可以通过 LED 的亮灭和闪烁状态，快速知道 AI 编程任务是否正在运行、是否有结果需要查看。

## 基本原理

VibeCodingLed 通过 USB 同时完成供电和通信。电脑上的 AI 工具读取配套 skill 后，会把当前 AI 编程任务状态发送给状态灯；状态灯内部芯片收到状态后，用板载 LED 显示结果。

LED 只有三种客户可见状态：

| LED 效果 | 含义 |
| --- | --- |
| 灯灭 | 当前没有 AI 编程任务在运行 |
| 闪烁 | AI 编程任务正在运行 |
| 常亮 | 有任务已经停止，需要查看结果或问题 |

常亮不一定代表失败，也可能表示某个任务已经完成。

## 你会收到什么

| 物品 | 作用 |
| --- | --- |
| VibeCodingLed 实物产品 | 插到电脑 USB 口，用 LED 显示 AI 编程状态 |
| VibeCodingLed skill | 给 AI 工具读取，用于自动完成设备检测、配置和状态监控 |

请保存完整 skill 文件夹。不要只保存其中某一个文件，否则 AI 工具可能无法完成自动配置。

## 使用步骤

1. 把 VibeCodingLed 插到电脑上能传数据的 USB 口。
2. 下载并解压随产品提供的 VibeCodingLed skill。
3. 打开 Codex、Cursor 或其它支持读取 skill 的 AI 编程工具。
4. 让 AI 工具读取 VibeCodingLed skill。
5. 对 AI 工具说：

```text
请读取 VibeCodingLed USB 状态灯 skill，帮我自动检测设备、配置 AI 编程状态监控，并验证灯灭、闪烁、常亮三种状态。
```

6. 等待 AI 工具完成配置，并按提示观察 LED 状态。

USB 口必须能通信，不只是供电。简单判断方法是：这个 USB 口平时能插 U 盘、键盘或鼠标。只用于充电的 USB 电源口不能使用。

## 如何判断已经正常工作

配置完成后，可以按下面现象判断：

| 现象 | 说明 |
| --- | --- |
| AI 告诉你设备已识别 | 电脑已经能和状态灯通信 |
| 没有 AI 编程任务时灯灭 | 当前没有任务运行 |
| AI 编程任务运行时灯闪烁 | 状态监控已经开始工作 |
| 有任务停止时灯常亮 | 有结果或问题需要你查看 |

如果插上后完全没有反应，先换一个能插 U 盘的数据 USB 口，再让 AI 重新读取 skill 检测。

## 简单演示

配置完成后，可以让 AI 做一次演示：

```text
请用 VibeCodingLed skill 做一次状态灯演示：先确认设备在线，然后让灯灭 3 秒、闪烁 5 秒、常亮 5 秒，最后恢复到当前 AI 监控状态。
```

你应该依次看到：灯灭、闪烁、常亮，然后回到当前 AI 工具的真实状态。

## 适用范围

当前优先支持 Windows 上的 Codex 和 Cursor 使用场景。其它 AI 编程工具如果支持读取 skill，或能让 AI 在任务开始、停止时触发本地状态更新，也可以接入。

VS Code、macOS、Linux 等环境的自动监控能力需要根据具体工具和系统继续适配；产品本身仍然通过 USB 与电脑通信。
